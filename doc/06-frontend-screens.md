# 프론트엔드 화면 구조

## 개요

ScaledTales Flutter 앱은 **Bottom Navigation Bar** 기반의 4개 메인 탭으로 구성되며, 각 탭은 독립적인 네비게이션 스택을 가집니다.

---

## 전체 구조

```
ScaledTales App
├── Splash Screen (초기 로딩)
├── Onboarding (첫 설치 시)
├── Authentication
│   ├── Login
│   └── Sign Up
└── Main App (Bottom Navigation)
    ├── Home Tab
    ├── Animals Tab
    ├── Logs Tab
    └── Settings Tab
```

---

## 네비게이션 패턴

### Bottom Navigation Bar (메인 4개 탭)

```dart
BottomNavigationBar(
  items: [
    BottomNavigationBarItem(
      icon: Icon(Icons.home),
      label: '홈',  // Home
    ),
    BottomNavigationBarItem(
      icon: Icon(Icons.pets),
      label: '개체',  // Animals
    ),
    BottomNavigationBarItem(
      icon: Icon(Icons.event_note),
      label: '일지',  // Logs
    ),
    BottomNavigationBarItem(
      icon: Icon(Icons.settings),
      label: '설정',  // Settings
    ),
  ],
)
```

## 1. 인증 플로우

### 1.1 스플래시 화면

**경로:** `/splash`

**목적:**
- 앱 초기화
- 자동 로그인 체크
- 온보딩 표시 여부 확인

**화면 구성:**
```
┌─────────────────────────────┐
│                             │
│                             │
│        [앱 로고]            │
│                             │
│      ScaledTales           │
│                             │
│    [로딩 인디케이터]         │
│                             │
└─────────────────────────────┘
```

**로직:**
1. 온보딩 완료 여부 확인
2. 자동 로그인 토큰 확인
3. 라우팅:
   - 온보딩 미완료 → `/onboarding`
   - 로그인 안 됨 → `/login`
   - 로그인 됨 → `/home`

---

### 1.2 온보딩 화면

**경로:** `/onboarding`

**목적:** 앱 주요 기능 소개

**화면 구성:**
```
┌─────────────────────────────┐
│  [슬라이드 1/4]              │
│                             │
│      [일러스트]              │

│                             │
│  [건너뛰기]        [다음] → │

**슬라이드 내용:**

2. 사육 일지

4. 브리딩 기록

**액션:**
- 건너뛰기 → `/login`
- 마지막 슬라이드 완료 → `/login`

## 3. 개체 탭

### 3.1 개체 목록 화면

**경로:** `/animals`

**AppBar:**
- 타이틀: "내 개체들"
- 액션: 검색 아이콘, 필터 아이콘

**화면 구성:**

#### 필터 칩 (상단)
```

```

#### 정렬 드롭다운
```
정렬: [최근 등록순 ▼]
```

#### 개체 리스트 (카드 형태)
```

│       Eublepharis macularius│
│       무게: 45g | 2살        │

│       무게: 38g | 1년 6개월  │

**액션:**
- 카드 탭 → 개체 상세
- [+] 버튼 → 개체 등록

### Riverpod 사용 패턴

| 패턴 | 사용 시점 |
|------|---------|
| `ConsumerWidget` | 로컬 상태 없이 ref만 필요한 화면 (대부분의 목록/상세 화면) |
| `ConsumerStatefulWidget` + `ConsumerState` | TextEditingController 등 로컬 상태와 ref를 함께 사용하는 Form 화면 (MeasurementFormScreen, CareLogFormScreen 등) |

**CRUD 호출 규칙:**
- Screen에서 `DioClient().dio`를 직접 호출하지 않는다.
- 반드시 `ref.read(xxxProvider(id).notifier).methodName(...)` 패턴으로 Notifier 메서드를 호출한다.
- 두 Provider 동기화 필요 시 `ref.invalidate(otherProvider)` 사용.

```dart
// 올바른 패턴
await ref.read(careLogListProvider(animalId).notifier).create(data);
ref.invalidate(allCareLogsProvider); // 전체 탭도 갱신

// 잘못된 패턴 (사용 금지)
await DioClient().dio.post(ApiConstants.careLogs(animalId), data: data);
```

### Role 기반 접근 제어 패턴

`userRoleProvider`를 통해 현재 사용자의 role을 구독하고, `canWrite` getter로 쓰기 권한을 판별한다.

```dart
// 위젯 build() 내 권한 체크
final canWrite = ref.watch(userRoleProvider).canWrite;

// FAB 조건부 표시
floatingActionButton: canWrite
    ? FloatingActionButton(onPressed: ..., child: Icon(Icons.add))
    : null,

// 버튼 조건부 표시
if (canWrite) FilledButton.icon(onPressed: ..., label: Text('추가')),

// CareLogCard 수정/삭제 콜백
CareLogCard(
  onEdit: canWrite ? () => _openForm(context, careLog: log) : null,
  onDelete: canWrite ? () => _confirmDelete(context, log) : null,
),
```

**적용 규칙:**
- `suspended` role: `canWrite = false` → 모든 생성·수정·삭제 버튼 비노출
- 나머지 role(`admin`, `seller`, `pro_breeder`, `user`): `canWrite = true` → 전체 기능 접근
- `userRoleProvider`는 로딩/오류 시 안전 기본값 `UserRole.user` 반환 (앱 정상 동작 보장)

## 네비게이션 플로우 다이어그램

```
로그인
  ↓
┌─────────────────────────────┐
│       Bottom Navigation      │
├──────┬──────┬──────┬─────────┤
│ 홈   │개체  │일지  │ 설정    │
└──────┴──────┴──────┴─────────┘

홈 탭:
  → 개체 등록 → 개체 상세
  → 일지 작성 → 일지 상세
  → 최근 개체 → 개체 상세

개체 탭:
  → 개체 등록 → 개체 상세
  → 개체 선택 → 개체 상세
       → 정보 탭
       → 측정 탭 → 측정 추가
       → 일지 탭 → 일지 작성

일지 탭:
  → 일지 작성 → 일지 상세
  → 일지 선택 → 일지 상세 → 수정

설정 탭:
  → 프로필 편집
  → 언어 설정
  → 테마 설정
  → 로그아웃 → 로그인
```
